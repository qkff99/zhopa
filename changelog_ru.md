## 0.6.2 — 2026-01-26

### Исправления
- Исправлен редкий вылет `pure virtual function call` в Debug HUD: убраны вызовы `:name()`/`alife_object()` для потенциально “протухших” server-объектов.
- Исправлено “залипание” после торговли: добавлен принудительный сброс `desired_position/desired_direction` и корректная обработка `FAILED/NO_SINK` с освобождением лидера/cleanup.
- Исправлена продажа: NPC больше не продают активное/экипированное оружие/броню и не продают последнюю пушку (гарантируется минимум один `wpn_`).
- Исправлен регресс, из-за которого мог “отваливаться” оффлайн и не срабатывали дампы: `safe_run` переведён на Lua 5.1 `unpack` и корректно обрабатывает `nil`-аргументы; хоткей дампа больше не зависит от прямого `ui_mcm.get`.

### Производительность
- Буферизация файлового логгера: снижены затраты на `io.open/io.close` за счёт пакетной записи.
- Добавлен TTL-кэш чтения настроек MCM (`get_config`) для снижения нагрузки при частых опросах.
- Оптимизировано вычисление уровня отряда при `sync_to_memory`: кэшируется имя уровня при неизменном `m_game_vertex_id`.
- Оптимизирован Debug HUD: фильтрация отрядов не на текущем уровне, пропуск обновлений при неизменном hint, cooldown на `try_register_brain`.
- Тюнинг глобального artifact scan: меньше “пустых” шагов и более крупные блоки обработки.

### Стабильность / Диагностика
- Добавлено `xpcall + debug.traceback` для ключевых SISKI апдейтов (brain/debug/scanner) и `brain:update()`: ошибки логируются с полным traceback и не должны валить игру из‑за SISKI.
- Усилены предохранители кэшей `perf` в memory (авто-сброс при аномальном росте).

## 0.6.1 — 2026-01-25

### Исправления
- MOVE теперь всегда пытается привязать отряд к smart через `assign_squad_to_smart` даже если у SIMBOARD нет стабильного резолва `name -> id` (добавлен единый резолвер через SIMBOARD → intel/memory).
- Убраны зависания MOVE: добавлен контроль прогресса по дистанции до цели, периодический rebind к smart при стагнации и деградация в `MOVE_TO_POS` при длительном “залипании”.
- Улучшена устойчивость `MOVE_TO_POS`: добавлены стадии деградации (retry/смартовый fallback) вместо немого force-complete; для offline отрядов `MOVE_TO_POS` конвертируется в smart-based MOVE к ближайшему смарту.
- Исправлен ретаргет задач: разрешён не только для `af_hunter` — учитываются срочность/приоритет и введён cooldown против “дрожания” целей (разные пороги для online/offline).
- Исправлена торговля: устранены предупреждения вида `SV: can't find child ... of parent ...` за счёт отказа от удаления предметов через release из инвентаря при отсутствии trader NPC; продажа теперь идёт через безопасный transfer в живого NPC на смарте (или откладывается, если “синка” нет).

### Новое
- Реализована полноценная система торговли между NPC.
- Кросс-уровневое перемещение: если целевой smart на другом уровне, сначала строится шаг к ближайшему level changer текущего уровня (`MOVE_TO_POS`), затем автоматически ставится финальная задача на целевой smart.
- Расширены метрики для диагностики движения: в memory и репортер/дебаг добавлены поля по stuck-состоянию MOVE и стадиям деградации `MOVE_TO_POS`, плюс “давность” ретаргета.
